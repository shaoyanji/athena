# Athena Architecture\n\n## Mission\n\nAthena is a portable workspace control plane for expressing authored truth in a small, explicit shape and projecting live workspace surfaces from that core.\n\n## Design principles\n\n- Prefer compact startup views over recursive discovery.\n- Keep authored truth small and legible.\n- Separate authored state from generated state.\n- Do not confuse projections with the system itself.\n- Make operator-facing surfaces explicit.\n- Prefer portable flake contracts over machine-local folklore.\n\n## Canonical shape\n\n### Portable canonical source\n\n- `github:shaoyanji/athena`\n\n### Canonical local working root\n\n- `projects/athena/`\n\n### Canonical live target\n\n- `/var/lib/nullclaw/workspace`\n\nThe flake is the portable source of truth.\nThe workspace root is the live materialization target.\n\n## Current structure\n\n### Authored core\n\n- `flake.nix`\n- `nix/default.nix`\n- `nix/registry/*`\n- `nix/taskfiles/*`\n- `src/*`\n\n### Live projections\n\n- workspace-root symlinks for `AGENTS.md`, `SOUL.md`, `USER.md`, `ATHENA.md`\n- `seed.json`\n- `registry/effective/registry.json`\n\n## Current execution model\n\nAthena currently supports four practical actions:\n\n### 1. Enter the dev shell\n\n```bash\nnix develop\n```\n\n### 2. Consume Athena as a public flake\n\n```bash\nnix develop github:shaoyanji/athena\n```\n\n### 3. Materialize the live workspace surface\n\n```bash\ncd /var/lib/nullclaw/workspace/projects/athena\nnix develop -c task -t taskfiles/Taskfile.yml activate\n```\n\nor\n\n```bash\nnix develop github:shaoyanji/athena --command athena-activate\n```\n\n### 4. Run health checks\n\n```bash\nnix develop -c task -t taskfiles/Taskfile.yml check\n```\n\n## Important distinction\n\nAthena is now more than a seed: it has a real live control-plane and materialization path.\n\nBut it still does not yet constitute a complete runtime bootstrap or production switch for the wider system.\nThe live plane is real; the total replacement boundary is not yet fully defined.\n\n## What is already real\n\n- public GitHub flake source\n- profile-backed dev shell surface\n- store-backed activation package\n- live workspace-root materialization with artifact-backed symlink proof\n- a wrapper-flake update path for fresh workspaces\n\n## Missing pieces for a full switch\n\nAthena is not a full runtime replacement until all of the following are explicit:\n\n1. startup contract — what reads Athena outputs, when, and why\n2. operator surface — the command or interface a human should trust as the primary entrypoint\n3. replacement boundary — which existing bootstrap or steering path Athena replaces\n4. verification path — proof that the surrounding system consumes Athena-authored outputs\n5. registry-driven profile/package/tool/skill surfaces that are real, not stubby\n\n## Intended end state\n\nAthena should become the canonical local context/control surface: small, portable, explicit, profile-oriented, and strong enough to guide weaker execution agents without forcing the human to reconstruct the whole system every session.\n
